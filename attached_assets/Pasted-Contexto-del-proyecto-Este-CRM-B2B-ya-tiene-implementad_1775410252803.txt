─────────────────────────────────────────────────────────────────────
OBJETIVO DE ESTE SPRINT
─────────────────────────────────────────────────────────────────────
Implementar el "Plan Comercial Opción 1 – Especialización" sobre el
código existente. Este plan divide el equipo en 3 roles funcionales
con métricas y flujos propios:

  Hunter  → genera leads (llama / visita). NO cotiza.
  Farmer  → recibe leads del Hunter y cierra ventas.
  Admin ventas → procesa cotizaciones en < 24 h.

  Flujo: Hunter genera lead → asigna a Farmer → Farmer pide cotización
         → Admin cotiza (< 24 h) → Farmer cierra.

  Metas a trackear:
  • Hunter: 20 llamadas/día (100/semana); 5 reuniones/semana (20/mes)
  • Farmer: responder lead en < 2 h; % de cierre; seguimientos activos
  • Admin: cotizaciones entregadas en < 24 h; volumen mensual

── 2. BACKEND — NUEVOS ENDPOINTS ──────────────────────────────────

a) PATCH /api/salespeople/:id → ya existe, extenderlo para aceptar
   y guardar functionalRole en el body.

b) PATCH /api/opportunities/:id → ya existe, extenderlo para aceptar
   hunterId, farmerId, y al cambiar status también actualizar
   stageEnteredAt = now().

c) GET /api/dashboard/commercial-plan
   Nuevo endpoint en artifacts/api-server/src/routes/dashboard.ts.
   Debe devolver un objeto con estas secciones calculadas desde BD:

   hunterMetrics: array por cada salesperson con functionalRole="hunter":
     { salespersonId, name,
       callsThisWeek: count activities WHERE type="call" AND salespersonId=X
         AND completedAt >= inicio de semana actual,
       callsWeekTarget: meta de goals (default 100 si no hay meta cargada),
       meetingsThisMonth: count activities WHERE type="visit" AND ... >= inicio de mes,
       meetingsMonthTarget: meta de goals (default 20),
       leadsGeneratedThisMonth: count opportunities WHERE hunterId=X
         AND createdAt >= inicio de mes }

   farmerMetrics: array por cada salesperson con functionalRole="farmer":
     { salespersonId, name,
       leadsAwaitingResponse: count opportunities WHERE farmerId=X
         AND status="new" AND stageEnteredAt < now() - interval 2 hours,
       avgResponseTimeHours: avg(stageEnteredAt - createdAt) en horas
         para oportunidades asignadas a este farmer,
       activeOpportunities: count WHERE farmerId=X AND status NOT IN (won,lost,closed),
       closeRate: count(won) / count(won+lost) * 100 para este farmer,
       staleOpportunities: count opportunities WHERE farmerId=X
         AND status NOT IN (won,lost,closed)
         AND id NOT IN (SELECT opportunityId FROM activities
           WHERE completedAt >= now() - interval 3 days
           AND opportunityId IS NOT NULL) }

   adminMetrics: array por cada salesperson con functionalRole="admin_ventas":
     { salespersonId, name,
       pendingQuotes: count opportunities WHERE status="quote_requested",
       avgQuoteTurnaroundHours: avg time between status change to
         "quote_requested" y status change to "quoted" (aproximar con
         updatedAt si no hay log de transición),
       quotesOnTimeRate: % de cambios a "quoted" que ocurrieron dentro
         de 24 h desde "quote_requested",
       quotesThisMonth: count opportunities WHERE status IN (quoted,negotiating,won)
         AND updatedAt >= inicio de mes }

   pipelineFunnel:
     { leadsNew, quoteRequested, quoted, negotiating, won, lost }
     counts de opportunities por status

d) GET /api/goals → lista metas (filtrable por salespersonId, period)
   POST /api/goals → crear meta (requireRole: admin, gerente)
   PATCH /api/goals/:id → editar meta (requireRole: admin, gerente)
   DELETE /api/goals/:id → eliminar (requireRole: admin, gerente)
   Crear archivo artifacts/api-server/src/routes/goals.ts y registrarlo
   en routes/index.ts.

e) REGLA DE NEGOCIO en PATCH /api/opportunities/:id:

   cambiar status a "quoted" o "quote_requested", responder 403 con
   mensaje: "Los Hunters no pueden cotizar. Solicitá la cotización al Admin."
